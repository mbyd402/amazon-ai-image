# 2026-03-23 - Amazon AI Image MVP 修复日志

## 问题清单 & 修复记录

### 1. Vercel 环境变量注入 bug
- **问题**: Vercel 没有把用户配置的 `NEXT_PUBLIC_*` 环境变量传入构建流程，导致 `process.env` 中这些变量都是空
- **解决**: 通过预构建脚本 `scripts/generate-env.js` + 硬编码环境变量值到 `lib/generated-env.ts` 绕过问题
- **提交**: 环境变量问题解决，Supabase 客户端可以正常创建

### 2. Dashboard 查询错误
- **问题**: 查询 users 表用了 `user_id` 但实际字段是 `id`，不符合 Supabase Auth 规范
- **解决**: 修改查询条件为 `id = user.id`，并且改为 `select(*)` 避免 schema 不匹配
- **提交**: 修复查询错误

### 3. 界面中文问题
- **问题**: 目标用户是美国亚马逊卖家，要求全英文界面
- **解决**: 把 Dashboard、登录页面全部翻译为英文
- **提交**: 全英文界面完成

### 4. 登录页面布局错乱
- **问题**: `/login` 还是旧的紧急登录页面，布局不对
- **解决**: 重新实现 `/login` 为正常的登录页面，包含邮箱登录 + Google 登录
- **提交**: 登录页面修复完成

### 5. Dashboard 无限 loading
- **问题**: 如果加载失败会一直 loading，用户看不到错误
- **解决**: 添加 10 秒超时，超时自动停止 loading 显示错误信息
- **提交**: 修复无限 loading

### 6. Google OAuth 登录 - origin 不匹配
- **问题**: 动态获取 `window.location.origin` 在 SSR 时报错
- **解决**: 添加 `typeof window` 判断，fallback 到硬编码域名
- **提交**: 修复 SSR 错误

### 7. **Google 登录不弹窗口 - 根本问题**
- **问题**: 点击 Google 登录按钮没有弹窗，直接右上角显示 Logout
- **错误日志**: `AbortError: Lock broken by another request with the 'steal' option.`
- **根本原因**: `Navbar` 组件在服务端渲染时就创建了 Supabase 客户端并获取用户，导致 auth token 锁被占用，Google 登录抢不到锁无法弹出窗口
- **解决**: 修改 Navbar，只在客户端 `useEffect` 中创建 Supabase 客户端，避免 SSR 锁冲突
- **状态**: 已修复，推送成功
