## 2026-03-20 - 亚马逊 AI 图片处理项目重要进展

### 核心问题解决：Supabase 免费版限制 + 广告拦截器

**根本原因分析：**
1. Supabase 免费版间歇性封锁 Vercel 的 outbound IP，导致客户端 `supabase.auth.getUser()` 超时
2. 服务端使用 `service_role key` 频繁调用触发风控和速率限制
3. 浏览器广告拦截器（uBlock Origin 等）阻止对 `supabase.co` 的请求

### 最终解决方案：

1. **客户端重试逻辑**：移除自动重试循环，改为用户手动重试按钮
2. **完全客户端化**：所有数据查询都在客户端使用 `anon key`
3. **绕过服务端存储权限问题**：API 返回 base64 图片而不是上传到 Supabase Storage
4. **清晰的用户提示**：明确指出广告拦截器可能阻止网站访问

### 关键技术实现：

1. **图片处理 API (`/app/api/process/route.ts`)**
   - 不再上传处理结果到 Supabase Storage
   - 直接返回 base64 编码的图片数据
   - 服务端只处理：AI API 调用、点数扣除、历史记录保存

2. **客户端处理 (`/components/ProcessPage.tsx`)**
   - 接收 base64 数据并转换为 data URL：`data:image/png;base64,...`
   - 大图预览，方便查看处理细节
   - 添加 + 号按钮，优化用户体验

3. **UI/UX 改进：**
   - 限制单次处理最多 5 张图片（从 10 张下调）
   - 预览网格尾部显示 + 号按钮，可追加图片
   - 显示当前图片数量：`(x/5)`
   - 清除的文件上传界面
