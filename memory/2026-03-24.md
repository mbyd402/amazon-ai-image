# 2026-03-24 - Amazon AI Image 项目修复日志

## 修复总结

### 问题
- Google OAuth PKCE 登录后，dashboard 崩溃，不输出日志，只显示 `Optimized dashboard loading...`
- `users` 表一直没有新用户数据，无法创建记录
- RLS 权限问题导致客户端无法创建用户

### 修复过程

#### 1. 第一次修复：Double guarantee 用户创建
- 服务端用 `service_role` 创建用户记录
- 客户端 fallback 用 RLS 允许创建
- PKCE flow 完全客户端处理，callback route 只做 redirect

#### 2. 第二次修复：把 client 从 useState 移到 useEffect
- useState factory 出错，React 崩溃前看不到日志
- 移到 useEffect 才能捕获错误，看到日志

#### 3. 第三次修复：逐步添加日志，从组件最开始就输出
- 发现还是只有 `Optimized dashboard loading...` 输出，说明还在更早之前崩溃
- 检查出语法错误，修复了缺失的 `)`

#### 4. 第四次修复（当前）：完全简化，所有初始化放到一个 useEffect
- 避免多次依赖 re-render，彻底消除 null 引用问题
- 每个步骤都加日志，方便调试
